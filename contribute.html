baoge AI 共建说明

如何手动更新这个网站

这个页面给你和后面一起共建的人看。不会代码也能改：找到文件，复制一段结构，换文字，刷新页面。

核心原则 页面结构可以学优秀网站，正文内容优先来自官方资料和我们自己的实践。涉及价格、套餐、模型、额度、地区时，只写核对方法，不写死承诺。

最快改法

你只要记住三个文件

首页

site_prototype/index.html

改导航、首页模块、官方资料、Codex 最新、交流群、案例库。

19节教程

site_prototype/tutorials/index.html

改每一节教程正文、步骤、作业、常见坑、来源。

样式

site_prototype/styles.css

改颜色、间距、卡片、表格、教程页排版。

图片

site_prototype/assets/

放二维码、截图、课程配图、教程示意图。

修改教程

怎么改某一节教程

第一步

打开 tutorials/index.html，搜索这一节的编号，例如 lesson-08。

第二步

只改这一节 <section id="lesson-08"> 到下一个 </section> 之间的内容。

第三步

每节尽量保留四块：准备什么、操作步骤、完成标准、常见坑。

第四步

刷新浏览器，看标题、列表、卡片有没有挤压或错位。

飞书导入

飞书文档怎么导入教程页

目前先走最稳的方式：飞书写内容，导出 Markdown，脚本导入网页。后面接入飞书 API 后，可以升级成给一个飞书链接自动导入。

第一步：在飞书里写

每一节单独写一个飞书文档。标题、步骤、常见问题、官方来源都写清楚。

第二步：导出 Markdown

在飞书文档里选择导出或复制为 Markdown。如果没有导出按钮，就先复制正文到本地 Markdown 文件。

第三步：保存到指定目录

保存到 content/tutorials/lesson-03.md 这种位置。文件名按课程编号来。

第四步：运行导入脚本

python3 site_tools/import_lesson.py content/tutorials/lesson-03.md

Markdown 文件必须带这一段头部

---
lesson: 03
kicker: 03 入门准备
title: 了解 Codex 桌面 App
lead: 这一节用一句话说明学完能做什么。
---

模板文件在：content/tutorials/lesson-template.md

官方来源

怎么给内容加来源

优先用这些来源

  • OpenAI Codex 官方页
  • OpenAI Developers Codex 文档
  • Codex Changelog
  • openai/codex GitHub Releases
  • ChatGPT Release Notes

写法

正文用自己的话讲，底部放来源链接。不要整段复制官方或第三方正文。

更新日期

每次改功能、权限、套餐、模型相关内容，都写“最后核对日期”。

不确定怎么办

写“以官方页面和账号实际显示为准”，不要说死。

共建规则

别人来更新时要守住什么

可以改

  • 教程步骤
  • 截图和配图
  • 官方来源
  • 案例和作业
  • 排障经验

不能直接放

  • 客户聊天原文
  • 个人隐私
  • 账号 Token
  • 未经确认的价格和套餐承诺
  • 第三方整段原文

账号类内容

只能写官方支持地区、ChatGPT/API Key 登录、团队合规路径和替代学习路径。不要写接码、虚拟卡、绕过地区限制或共享违规账号。

推荐格式

每次更新写清：改了哪一节、依据哪个来源、最后核对日期、还有什么不确定。

最终目标

让这个站能被大家持续补充，但风格、结构和可信度不散。
