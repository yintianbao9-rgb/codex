05 入门准备

连接第三方 API 与工具

Codex 真正有价值的地方，是把浏览器、飞书、GitHub、知识库、设计工具连接成流程。

先连哪些

  • GitHub：公开站共建、版本管理、Pages 发布。
  • 飞书：日报、文档、知识库和表格。
  • 浏览器：查看网页、验证 UI、抓公开资料。
  • 本地文件夹：课程、脚本、图片和案例库。

连接原则

  1. 先只读，再允许写。
  2. 先练习文件夹，再真实项目。
  3. 先人工确认，再自动发送。
  4. 权限越大，日志越要清楚。

baoge AI 例子

客户问题从微信/飞书进来，先汇总成日报，再变成选题、口播脚本、教程 FAQ，最后沉淀到公开站。

完成标准

你能画出自己的工具连接图：输入从哪里来、AI 做什么、结果去哪里、谁来确认。
